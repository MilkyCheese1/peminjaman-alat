TrustEquip.id

TrustEquip.id adalah aplikasi web untuk mengelola peminjaman alat, kategori alat, data pengguna, pengembalian, notifikasi, laporan, dan log aktivitas dalam satu sistem yang terpusat. Aplikasi ini berbasis SPA, jadi tampilan halaman dibuat dari resources/js dan data diambil melalui API Laravel.

Ringkasan Sistem

Apa yang dilakukan aplikasi ini

Sistem ini fokus pada alur peminjaman alat dari sisi admin, owner, staff, dan peminjam. Aplikasi menyimpan data utama di database, lalu menampilkan dan mengelola semuanya melalui antarmuka Vue yang responsif.

Inventaris

Mengelola alat dan kategori

Admin dapat menambah, mengubah, dan menghapus data alat serta kategori. Owner biasanya memakai data ini untuk memantau aset, sedangkan peminjam hanya melihat daftar alat yang tersedia.

Transaksi

Mengatur peminjaman dan pengembalian

Staff menangani transaksi peminjaman, persetujuan, dan konfirmasi pengembalian. Sistem juga menghitung denda berdasarkan tanggal dan kondisi barang.

Kontrol

Memantau aktivitas dan notifikasi

Admin dan owner dapat melihat notifikasi, log aktivitas, dan statistik agar keputusan operasional lebih mudah dipantau.

Relasi Antar Tabel

Bagaimana tabel saling terhubung

Relasi utama dibuat agar data tidak dobel dan proses peminjaman tetap konsisten. Beberapa tabel juga menyimpan snapshot teks supaya riwayat lama tidak berubah ketika data master diubah.

Relasi inti

categories.id → tools.category_id

Kategori menjadi induk alat. Satu kategori bisa punya banyak alat.

users.id → borrowings.peminjam_id

Data peminjam terhubung ke transaksi peminjaman.

users.id → borrowings.petugas_id

Staff atau petugas yang memproses transaksi tercatat sebagai relasi ke user.

tools.id → borrowings.alat_id

Transaksi menunjuk ke alat yang dipinjam.

Relasi pendukung

users.id → activity_logs.user_id

Log aktivitas menyimpan siapa yang melakukan aksi.

users.id → notifications.user_id

Notifikasi dikirim ke user tertentu.

borrowings menyimpan snapshot

Kolom seperti nama peminjam, nama alat, kategori, dan petugas_nama disimpan ulang supaya riwayat transaksi tetap sama walau data master berubah nanti.

feedback_entries berdiri sendiri

Feedback dari landing page tidak punya foreign key karena hanya menyimpan ulasan publik.

Alasan Tipe Data

Mengapa tipe data itu dipakai

Inti pemilihannya sederhana: angka kecil pakai tipe kecil, tanggal pakai tipe tanggal, teks bebas pakai text, dan path file disimpan sebagai string. Ini membuat struktur database tetap rapi dan efisien.

Tipe numerik

int unsigned dipakai untuk ID, harga, dan total denda karena datanya positif dan bisa tumbuh besar. tinyint unsigned cocok untuk role, status, dan kondisi karena nilainya hanya beberapa pilihan tetap. smallint unsigned cocok untuk stok karena lebih hemat daripada integer besar.

Tipe teks

varchar dipakai untuk data pendek dan terstruktur seperti nama, email, kode, atau path file. text dipakai untuk alamat, deskripsi, dan laporan karena isinya bisa panjang dan tidak selalu bisa diprediksi.

Tipe waktu dan biner

date dipakai jika yang penting hanya tanggal. timestamp dipakai untuk audit waktu buat dan update data. varbinary(16) dipakai untuk IP address agar bisa menyimpan IPv4 dan IPv6 dengan lebih rapi.

File pendukung penting

Router utama ada di resources/js/router/index.js. Layout sidebar role ada di resources/js/components/layout. Komponen pengelolaan data generik ada di resources/js/components/admin/CrudTablePage.vue. Endpoint API ada di routes/api.php.

Catatan struktur

Di proyek ini tidak ada file auth/login.blade.php seperti aplikasi Blade tradisional. Karena front-end memakai Vue SPA, login dan register berada di page Vue, lalu berkomunikasi dengan controller API Laravel.

Catatan stabilitas: form yang mengunggah file sekarang dipusatkan lewat resources/js/lib/api.js. Jika request memakai FormData dengan metode PUT atau PATCH, helper akan mengubahnya menjadi POST dan menambahkan _method secara otomatis. Pola ini dipakai agar upload file tetap terbaca konsisten oleh Laravel dan PHP.
Daftar Fitur

Penjelasan setiap fitur

Landing page

Menampilkan identitas aplikasi, manfaat utama, daftar alat, workflow peminjaman, testimoni, dan form feedback publik.

Login dan registrasi

User bisa masuk dengan email dan password atau mendaftar sebagai peminjam baru. Setelah login, session disimpan di localStorage.

Manajemen user

Admin dapat membuat, mengubah, dan menghapus pengguna, mengatur role, status, profil, dan foto akun.

Manajemen alat

CRUD alat, stok, kondisi, status, lokasi, harga asli, dan gambar alat agar inventaris selalu mutakhir.

Manajemen kategori

CRUD kategori alat seperti Elektronik, Mekanik, Safety, atau kategori lain sesuai kebutuhan gudang.

Manajemen peminjaman

Staff bisa membuat transaksi, mengubah status, menambahkan bukti, dan menjaga riwayat peminjaman tetap rapi. Upload file memakai pola aman melalui helper API.

Pengembalian

Staff mengonfirmasi status pengembalian, kondisi barang, dan sistem menghitung denda secara otomatis berdasarkan tanggal dan harga alat. Bukti pengembalian juga disimpan lewat alur upload yang kompatibel dengan Laravel.

Laporan

Menampilkan ringkasan transaksi dan data operasional untuk admin, staff, atau owner.

Notifikasi

Memberi info, warning, success, atau error kepada user berdasarkan aktivitas dan kondisi sistem.

Log aktivitas

Mencatat siapa melakukan apa, pada entitas apa, dan kapan aktivitas itu terjadi.

Akun dan profil

User dapat melihat dan memperbarui profil, avatar, dan password dari halaman akun masing-masing.

Feedback publik

Pengunjung landing page dapat memberi ulasan dan rating yang kemudian ditampilkan pada bagian testimoni.

Role & Hak Akses

Pembagian role pengguna

Pembagian akses mengikuti menu sidebar dan route yang tersedia pada router. Ini membuat setiap role fokus pada tugasnya sendiri tanpa membuka data yang tidak perlu.

Admin
  • Akses paling lengkap ke dashboard dan data master.
  • Kelola user, alat, kategori, peminjaman, notifikasi, dan log aktivitas.
  • Mengelola akun serta melihat laporan administrasi.
Owner
  • Melihat statistik dan ringkasan operasional.
  • Akses alat, peminjaman, notifikasi, dan akun owner.
  • Fokus pada monitoring dan pengambilan keputusan.
Staff
  • Menangani peminjaman, pengembalian, laporan, notifikasi, dan akun staff.
  • Bertugas sebagai operator harian di proses transaksi.
  • Dapat memeriksa kondisi barang dan menghitung denda.
Peminjam
  • Mengakses daftar alat, riwayat peminjaman, notifikasi, dan akun pribadi.
  • Melihat status pengajuan serta perkembangan transaksi miliknya.
  • Role ini adalah akun default saat registrasi baru.
Berdasarkan router, halaman publik hanya mencakup landing, login, dan register. Setelah autentikasi, user diarahkan ke dashboard sesuai role melalui resources/js/auth/session.js.

Tabel yang sudah tidak dipakai

Dampaknya

Database jadi lebih ramping dan hanya menyimpan data inti aplikasi. Login, register, peminjaman, notifikasi, laporan, dan log tetap jalan karena semuanya bergantung pada tabel aktif yang masih tersisa.
